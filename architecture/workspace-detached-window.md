SH-21: 複数モニタ向けWorkspace別ウィンドウ表示

概要:
- 単一インスタンス前提を維持したまま、選択したWorkspaceを別ウィンドウで表示できるようにする。
- メインShellはタブ・メニュー・保存・セッション管理の親として残す。
- 別ウィンドウは NoteNest / IdeaNest / ChatNest の作業画面を表示する補助ウィンドウとする。
- 複数モニタで、NoteNestをメイン、ChatNestを別画面、IdeaNestを別画面のように配置できる状態を目指す。

初期方針:
- 別プロセス・別インスタンス化はしない。
- 同一プロセス内の追加Windowとして扱う。
- タブは管理単位としてShell側に残す。
- 分離中タブは「別ウィンドウ表示中」と分かるプレースホルダ表示にする。
- 別ウィンドウを閉じた場合は、WorkspaceをShellタブ内表示へ戻す。
- 最初の実装対象はNoteNest Workspaceのみでもよい。
- セッションへの分離状態保存は初期実装では対象外。

注意点:
- WPFのViewは同時に2つの親へ所属できないため、Viewの移動ではなく、別Window用Viewを生成して同じWorkspaceViewModelへバインドする方式を優先する。
- 保存、未保存確認、Ctrl+S、タブ閉鎖、Window閉鎖時の責務境界を事前に整理する。
- TempNestは分離対象外とする。
- 保存形式・セッション形式は初期実装では変更しない。

想定フェーズ:
1. backlog追記
2. 設計メモ作成
3. NoteNestのみ最小試作
4. IdeaNest / ChatNestへ展開可否判断
5. セッション復元対象にするか判断

優先度:
- 大型UX改善候補
- 複数モニタ利用者には効果が大きい
- 状態管理が複雑なため、即実装ではなく設計メモを挟む
