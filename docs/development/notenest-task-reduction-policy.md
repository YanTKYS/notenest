# NoteNest タスク管理縮退方針

> 作成: v2.13.0 TD-52  
> 目的: NoteNest の今後の役割と、タスク管理機能の縮退方針を明記する。

---

## NoteNest の主目的（v2.13.0 以降）

NoteNest は、プロジェクト単位でノートを整理し、  
本文中のマーカーとノート間リンクから作業ポイントを見つける Workspace とする。

本格的なタスク管理は主目的にしない。

---

## 残す主機能

以下は NoteNest の中心機能として維持する。

- ノートブック
- ノート
- 本文エディタ
- 本文中のマーカー自動抽出（TODO / FIXME / NOTE 等）
- マーカー一覧
- ノート間リンク
- Markdown エクスポート
- 検索

---

## 縮退候補（今回は削除しない）

以下は今後の縮退候補として整理する。現時点では削除しない。

- タスクの手動作成
- タスク完了管理
- タスクコメント
- タスク関連付け
- タスク中心の右ペイン UI
- タスクを独立した管理対象として扱う導線

---

## 本文マーカー管理への寄せ方

作業ポイントは、本文から抽出されるマーカーとして扱う。

- `TODO` / `FIXME` / `NOTE` などの既存マーカーを中心に扱う
- タスクを別オブジェクトとして管理するのではなく、本文に書いた作業ポイントを一覧で見つける方向へ寄せる

---

## 既存データ互換

既存の `.notenest` データは壊さない。

- 既存タスク情報を削除しない
- 既存タスク情報を読めなくしない
- 保存形式を変更しない
- schema bump しない（NoteNest schema `1.4.1` を維持する）

将来 UI 上でタスク導線を下げる場合も、既存データ互換を優先する。

---

## 今後の段階案

| 段階 | 内容 |
|------|------|
| 第1段階（完了 v2.13.0） | 方針整理。NoteNest は本格的なタスク管理を主目的にしないと明記する |
| 第2段階（完了 v2.13.1） | 右ペインでマーカー / リンク TabControl を先頭に移動し、タスクを下部の補助セクションへ下げた。タスク機能・データは削除しない |
| 第3段階（完了 v2.13.4 M16） | タスク管理UIを、通常利用の主導線から既存データ互換の表示へ縮退した |
| 第4段階 | 保存形式変更を伴う削除・移行は、必要性が明確になるまで実施しない |

**v2.13.2（L17・L19）補足**: 第2段階の延長として、右ペイン開閉ボタンのツールチップを開閉状態に連動させ、編集エリアに現在のノートブック名を表示した。マーカー / リンク中心の Workspace で「今どういう状態か」「どこを見ているか」を見失いにくくする補助 UI であり、タスク機能・データの変更は伴わない。

**v2.13.4（M16）補足 — タスク管理UIの互換表示化**:

- 新規作業管理は、タスク作成ではなく本文の `TODO` / `FIXME` / `NOTE` マーカーで行う方針とした。
- タスク新規作成導線（右ペイン上部の「タスクを追加」ボタン・各グループの「+」ボタン）を通常UIから外した。`AddTaskCommand` / `AddTaskMenu_Click` 自体は削除していない（既存コードへの影響を避けるため）。
- 既存タスクが1件もない場合は、タスク欄を大きく表示せず「作業ポイントは本文の TODO / FIXME で管理できます。」の一言のみを表示する。
- 既存タスクが1件でもある場合は、「タスク（互換）」として引き続き確認できる。タスクが存在しないグループ（今日／今週／バックログ）は表示しない。
- 既存タスクの完了切替・コメント編集・関連ノート付け替え・削除・移動などの既存操作は変更していない。
- 保存形式・NoteNest schema（`1.4.1`）・保存処理・読込処理は変更していない。

**v2.13.5（M16 フォローアップ）補足 — タスク欄が空の場合の右ペイン領域縮小**:

- v2.13.4 時点では、既存タスクが0件でも右ペイン下段の `RowDefinition` が `2* / MinHeight=100` のまま固定されており、ヒント文だけの表示でも一定の面積を占有し続けていた（レビュー指摘）。
- `HasNoTasks` に応じて下段の `Height`/`MinHeight` を切り替え、タスクが0件の場合はヒント文の高さのみに縮め、マーカー / リンク領域（上段）へ余った領域を譲るようにした。
- 既存タスクが1件でもある場合の表示・挙動（`2* / MinHeight=100`、タスク（互換）表示、既存操作）は変更していない。
