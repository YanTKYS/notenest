# NoteNest タスク管理縮退方針

> 作成: v2.13.0 TD-52  
> 目的: NoteNest の今後の役割と、タスク管理機能の縮退方針を明記する。

---

## NoteNest の主目的（v2.13.0 以降）

NoteNest は、プロジェクト単位でノートを整理し、  
本文中のマーカーとノート間リンクから作業ポイントを見つける Workspace とする。

本格的なタスク管理は主目的にしない。

---

## 残す主機能

以下は NoteNest の中心機能として維持する。

- ノートブック
- ノート
- 本文エディタ
- 本文中のマーカー自動抽出（TODO / FIXME / NOTE 等）
- マーカー一覧
- ノート間リンク
- Markdown エクスポート
- 検索

---

## 縮退候補（今回は削除しない）

以下は今後の縮退候補として整理する。現時点では削除しない。

- タスクの手動作成
- タスク完了管理
- タスクコメント
- タスク関連付け
- タスク中心の右ペイン UI
- タスクを独立した管理対象として扱う導線

---

## 本文マーカー管理への寄せ方

作業ポイントは、本文から抽出されるマーカーとして扱う。

- `TODO` / `FIXME` / `NOTE` などの既存マーカーを中心に扱う
- タスクを別オブジェクトとして管理するのではなく、本文に書いた作業ポイントを一覧で見つける方向へ寄せる

---

## 既存データ互換

既存の `.notenest` データは壊さない。

- 既存タスク情報を削除しない
- 既存タスク情報を読めなくしない
- 保存形式を変更しない
- schema bump しない（NoteNest schema `1.4.1` を維持する）

将来 UI 上でタスク導線を下げる場合も、既存データ互換を優先する。

---

## 今後の段階案

| 段階 | 内容 |
|------|------|
| 第1段階（完了 v2.13.0） | 方針整理。NoteNest は本格的なタスク管理を主目的にしないと明記する |
| 第2段階（完了 v2.13.1） | 右ペインでマーカー / リンク TabControl を先頭に移動し、タスクを下部の補助セクションへ下げた。タスク機能・データは削除しない |
| 第3段階（M16） | 既存タスク機能は互換目的で維持しつつ、新規利用を促さない導線へ整理する |
| 第4段階 | 保存形式変更を伴う削除・移行は、必要性が明確になるまで実施しない |
