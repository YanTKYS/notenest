## 旧UI棚卸しにより変更した項目（TD-36 / v2.11.5）

v2.10.21 / SH-25 でメニューバー直下の上部バー（`NestSuite / NoteNest / IdeaNest / ChatNest` ボタン行）が削除された。  
しかしチェックリストの「4.5. ツールランチャー表示確認」セクションが旧UI前提のままだったため、現行UIに合わせて修正した。

### 削除した確認項目（旧ツールランチャー UI — v2.10.21 以降存在しない）

- `ヘッダーバー左に「NestSuite」ラベルが表示される`
- `ヘッダーバー右側に NoteNest / IdeaNest / ChatNest のボタンが横並びで表示される`
- `左側のツール選択パネルが表示されない（Workspace が全幅）`
- `NoteNest ボタンをクリックすると NoteNest タブへ移動または新規作成される`
- `IdeaNest ボタンをクリックすると IdeaNest タブへ移動または新規作成される`
- `ChatNest ボタンをクリックすると ChatNest タブへ移動または新規作成される`
- `アクティブなタブのツールに対応するボタンがハイライト表示される`
- `TempNest 選択中はツールランチャーボタン（NoteNest/IdeaNest/ChatNest）のハイライトがなくなる`

### 置き換えた確認項目（現行ツールメニュー UI）

セクション名を「ツールメニュー確認」に変更し、以下に置換:

- `ツールメニューから NoteNest タブへ移動または新規作成できる`
- `ツールメニューから IdeaNest タブへ移動または新規作成できる`
- `ツールメニューから ChatNest タブへ移動または新規作成できる`
- `アクティブなタブに対応するツールメニュー項目にチェックマークが付く`
- `TempNest 選択中はツールメニューの NoteNest / IdeaNest / ChatNest のチェックマークがすべて外れる`

## チェック項目を補足文書へ移した理由（TD-35）

v2.11.4 以前、チェックリスト冒頭に全バージョン変更履歴を一行の長文として記述していた。  
これにより、毎回確認すべき項目が長い背景説明の後に並ぶ状態となり、認知負荷が上がっていた。

以下の方針で整理した。

- **常用チェックリスト本文**: リリースのたびに確認する機能・品質・バージョン確認項目を残す
- **本ファイル（補足）**: バージョン別の追加理由・背景説明・経緯を移す
- **既知の非ブロッキング警告**: 本ファイルに記載する（常用チェック項目にしない）

## 既知の警告・注意事項（非ブロッキング）

### `MainWindowEventBoundaryTests.PrivateInstance` warning

テスト実行時に `xunit.assert` の内部実装に関する警告が出る場合がある。  
テスト結果には影響しない。ブロッキングではない。

### Node.js warning

CI 環境の Node.js バージョンに関する deprecation warning が出る場合がある。  
GitHub Actions の `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: 'true'` 設定で対処済み。

## net48_test について

v2.7.0〜v2.7.2 で .NET Framework 4.8 テストビルドを追加・検証したが、単一 EXE 配布方針（RJ-5）と衝突するため v2.7.3 で通常リリース成果物から除外した。  
`NestSuite.Net48Test/` は検証履歴として保持しているが、通常 build/test 対象ではない。  
詳細は backlog の RJ-1 を参照。
