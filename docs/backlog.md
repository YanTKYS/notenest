# Backlog

## 将来検討：マーカー行の表示／非表示

本文中の `[TODO]` `[FIXME]` `[NOTE]` を含む行を、一時的に非表示にする要望がある。

ただし、v0.1.2 時点では実装しない。

### 理由

- WPF 標準 TextBox では、行単位の非表示を安全に実装しにくい
- 表示用テキストと保存用本文の不整合により、本文消失リスクがある
- カーソル位置、検索、マーカークリック移動との整合が複雑になる
- v0.1.x 段階では、ノート本文の安全性と保存安定性を優先する

### 将来的に検討する場合の前提

- 非表示中は編集不可にする
- 表示用ビューと保存用本文を明確に分離する
- TextBox ではなく、行単位表示に適したエディタ部品を採用する
- マーカー行を非表示にするのではなく、マーカー一覧側のフィルタ機能で代替する
