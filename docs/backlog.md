## v1.2.0以降候補：中難易度

既存 UI・設定保存・複数ペイン構成に影響するため、事前設計が必要な項目です。

- **右ペインの折り畳み**
  - タスク・マーカー領域をまとめて非表示にし、中央エディタを広く使えるようにする
  - 折り畳み状態の記憶については「画面レイアウトを記憶」と合わせて設計する

- **画面レイアウトを記憶**
  - ウィンドウサイズ、左ペイン幅、右ペイン幅、右ペイン折り畳み状態を次回起動時に復元する
  - テーマ設定との保存形式統合が必要（現在は `.notenest` の `settings` に一部保存）

- **Markdown 形式でのエクスポート**
  - 既存 `.txt` 出力に追加。ヘッダー整形・リスト表現の仕様定義が必要
  - 現在のノート本文は Markdown 準拠ではないため、出力仕様の設計から始める

- **HTML 形式でのエクスポート**
  - 印刷・閲覧向け。CSS テンプレート設計が必要
  - Markdown エクスポートの実装後に検討するとよい

- **タスク一覧・タスクコメントのエクスポート**
  - 現在のテキストエクスポートはノート本文のみ。タスクの出力フォーマット設計が必要

- **マーカー一覧のエクスポート**
  - エクスポート時にノート本文を再スキャンして抽出する形を想定
  - 出力フォーマット設計が必要

- **自動保存**
  - 一定間隔での背景保存。手動保存ダイアログとの競合・`*` 表示との整合に注意

- **検索の正規表現対応**
  - 現在は単純文字列検索。正規表現モード追加は既存検索ダイアログの拡張

- **ノートリンク補完**
  - `[[` 入力時にノート一覧をサジェストするインライン補完
  - TextBox ベースでは実装難易度が上がる可能性がある

- **ノート間リンクのノートブック名修飾**
  - `[[ノートブック名/ノート名]]` 形式での一意指定
  - リンク解決ロジックの変更と、挿入ダイアログへの反映が必要

- **リンク切れ一覧**
  - 存在しないノート名へのリンクを一覧化する
  - 全ノートスキャンとリンク解決ロジックを組み合わせた実装が必要

## 将来検討：高難易度

エディタ内部構造や既存設計に大きく影響するため、長期的な検討が必要な項目です。

- **編集箇所の行番号ハイライト**
  - 現在編集中の行番号を視覚的に強調する
  - 行番号表示（v0.5.0 で実装済み）を前提とした機能
  - WPF 標準 TextBox ではキャレット位置と行番号ガターの同期が複雑になる

- **バックリンク一覧**
  - 指定ノートを参照している `[[...]]` リンクの逆引き表示
  - 全ノートスキャンと参照管理の仕組みが必要

- **ノートリンクの視覚的ハイライト**
  - エディタ内の `[[ノート名]]` を色付き表示する
  - WPF 標準 TextBox では安全な実装が難しく、エディタ部品の差し替えが前提

- **マーカー行の表示／非表示**
  - `[TODO]` `[FIXME]` `[NOTE]` を含む行を一時的に非表示にする
  - WPF 標準 TextBox では特定行の安全な非表示が難しい
  - 表示用テキストと保存用本文の乖離リスクがある
  - まずマーカー一覧側のフィルタ機能（実装済み）で代替を優先する

## 対象外・当面見送り

NoteNest の設計方針から意図的に除外しているもの。要望があっても原則実装しません。

| 機能 | 理由 |
|------|------|
| 画像貼り付け | 軽量テキスト管理ツールの軸がぶれる。単一 JSON への画像埋め込みはファイルサイズ増大を招く |
| 共同編集 | ローカル単一ファイル管理の方針と根本的に合わない。排他制御・マージ処理が複雑 |
| クラウド同期 | ローカル利用を前提としている。OneDrive 等のフォルダへ手動配置で代替可能 |
| タスク期限・優先度 | モデル拡張だけでなく、表示・通知・ソート設計が広範に必要 |
| 通知機能 | タスク期限が前提。デスクトップ通知の OS 依存があり方針未定 |
| 高機能 Markdown エディタ化 | エディタ部品の差し替えは影響範囲が大きい。安定性優先のため見送り |
| Markdown プレビュー | WebView2 等の依存が増える。標準 TextBox 方針と整合しない |
| シンタックスハイライト | 高機能エディタ部品が前提 |
| 添付ファイル管理 | 単一 JSON ファイルとの相性が悪い |
| バックアップ自動化 | `.notenest` ファイルのコピーで代替可能。アプリ内実装は過剰 |
| 複数プロジェクトのタブ管理 | 多重起動で代替可能 |
| 文字数表示 | 現時点の主要価値（プロジェクト管理）に対して優先度が低い |
| Git 連携 | `.notenest` ファイルをコミット対象とすれば外部ツールで完結 |
