## v1.0.0 以降で検討

実装すれば便利だが、安定版到達後に判断する候補。

| 項目 | 概要 |
|------|------|
| Markdown 形式でのエクスポート | 既存 `.txt` 出力に追加。ヘッダー整形・リスト表現の仕様定義が必要 |
| HTML 形式でのエクスポート | 印刷・閲覧向け。CSS テンプレート設計が必要 |
| バックリンク一覧 | 指定ノートを参照している `[[...]]` リンクの逆引き表示 |
| リンク切れ一覧 | 存在しないノート名へのリンクの一覧化 |
| ノートリンク補完 | `[[` 入力時にノート一覧をサジェスト |
| ノート間リンクのノートブック名修飾 | `[[ノートブック名/ノート名]]` 形式での一意指定 |
| タスク一覧・タスクコメントのエクスポート | 現在は対象外。出力フォーマット設計が必要 |
| マーカー一覧のエクスポート | 出力時に再抽出する形を想定 |
| 自動保存 | 一定間隔での背景保存。保存ダイアログとの競合に注意 |
| 検索の正規表現対応 | 現在は単純文字列検索 |
| ノートリンクの視覚的ハイライト | エディタ部品の差し替えが前提 |
| マーカー一覧のソート切替 | 種別順/ノート順/行番号順 |

## 当面見送り

NoteNest の設計方針から意図的に除外しているもの。当面実装しません。

| 機能 | 理由 |
|------|------|
| 画像貼り付け | 軽量テキスト管理ツールの軸がぶれる。単一 JSON への画像埋め込みはファイルサイズ増大を招く |
| 共同編集 | ローカル単一ファイル管理の方針と根本的に合わない。排他制御・マージ処理が複雑 |
| クラウド同期 | ローカル利用を前提としている。OneDrive 等のフォルダへ手動配置で代替可能 |
| タスク期限・優先度 | モデル拡張だけでなく、表示・通知・ソート設計が広範に必要 |
| 通知機能 | タスク期限が前提。デスクトップ通知の OS 依存があり方針未定 |
| 高機能 Markdown エディタ化 | エディタ部品の差し替えは影響範囲が大きい。安定性優先のため見送り |
| Markdown プレビュー | WebView2 等の依存が増える。標準 TextBox 方針と整合しない |
| シンタックスハイライト | 高機能エディタ部品が前提 |
| 添付ファイル管理 | 単一 JSON ファイルとの相性が悪い |
| バックアップ自動化 | `.notenest` ファイルのコピーで代替可能。アプリ内実装は過剰 |
| 複数プロジェクトのタブ管理 | 多重起動で代替可能 |
| 文字数表示 | 現時点の主要価値（プロジェクト管理）に対して優先度が低い |
| Git 連携 | `.notenest` ファイルをコミット対象とすれば外部ツールで完結 |
| マーカー行の表示／非表示 | WPF 標準 TextBox では安全に実装できない。詳細は本ドキュメント下部参照 |

## 将来検討：マーカー行の表示／非表示

本文中の `[TODO]` `[FIXME]` `[NOTE]` を含む行を一時的に非表示にする要望がある。

**現時点では実装しない。** 理由は以下のとおり。

- WPF 標準 TextBox では、特定行だけを安全に非表示にする実装が難しい
- 表示用テキストと保存用本文が乖離し、本文消失リスクがある
- カーソル位置・検索・マーカークリック移動との整合が複雑になる

**将来的に検討する場合の前提：**

- 非表示中は編集不可にする
- 表示用ビューと保存用本文を明確に分離する
- TextBox ではなく、行単位表示に適したエディタ部品を採用する
- マーカー行を非表示にするのではなく、マーカー一覧側のフィルタ機能で代替することを優先する
