# テストシナリオ — NoteNest v1.5.4

## v1.5.4 確認観点まとめ

v1.5.1 時点で動作を確認すべき機能の一覧です。各詳細シナリオは下記セクションを参照してください。
v1.4.6 では v1.4.x の変更後に主要機能が回帰していないことを総合確認します（§41）。
v1.4.5 では MainWindow partial群のイベント整理後も、ショートカット、右クリック、ドラッグ＆ドロップ、ダイアログ起動が回帰していないことを確認します（§40）。
v1.4.0 では新機能を追加せず、v1.3.x の責務分離後に既存機能が壊れていないことを総合確認します（§35）。
v1.3.6 ではプロジェクトセッションとライフサイクルを分離し、責務別 Coordinator への分割後も変更分類を維持していることを確認します（§34）。
v1.3.5 では責務間のイベント調停を Coordinator へ分離し、データ変更と内部同期の分類を確認します（§33）。
v1.3.4 ではエディタ状態と保存モデル変換を独立コンポーネントへ移しました（§32）。
v1.3.3 では責務分離の第二段階として状態所有者を独立ViewModelへ移しました（§31）。
v1.3.2 では内部構造の保守性改善に加え、アプリケーションバージョンと保存スキーマバージョンを分離しました（§30）。
v1.3.1 では左ペインのプロジェクト表示名を改善しました（§29）。
v1.3.0 ではタイトルバーへのバージョン表記を追加しました（§28）。
v1.2.6 では起動時スタートダイアログを追加しました（§27）。
v1.2.5 ではファイル関連付け起動（コマンドライン引数からのファイルオープン）に対応しました（§26）。
v1.2.4 ではヘルプメニューからチュートリアル画像を表示できるようになりました（§25）。

## §20 v0.9.0 リリース前総点検

### §20-1 保存・読込の基本回帰

1. 新規プロジェクトを作成し、ノートブック・ノート・タスクを 1 件ずつ追加する
2. 名前を付けて保存（例：`test.notenest`）する
3. NoteNest を一旦終了して再起動する
4. 同ファイルを開き、追加した内容が復元されることを確認する
5. 内容を編集して `Ctrl+S` で上書き保存する
6. 同フォルダに `test.notenest.bak` が作成されていることを確認する
7. `test.notenest.tmp` が残っていないことを確認する

### §20-2 保存のキャンセル

1. 「名前を付けて保存」ダイアログをキャンセルする
2. アプリがクラッシュせず、未保存マーク（`*`）が維持されることを確認する
3. 保存先フォルダにファイルが作られていないことを確認する

### §20-3 `.bak` からの復元

1. プロジェクトを保存し、`.bak` が作られた状態にする
2. NoteNest を終了する
3. エクスプローラで `.notenest` 本体をテキストエディタで開き、JSON を意図的に破損させる（例：先頭の `{` を削除）
4. NoteNest を起動して破損ファイルを開こうとし、エラーメッセージが表示されることを確認する
5. 破損 `.notenest` を別名へ退避し、`.bak` を `.notenest` にリネームする
6. 再度 NoteNest で開き、`.bak` 時点の内容で復元できることを確認する

### §20-4 不正 JSON・空ファイルの読込

1. 適当なテキストエディタで空ファイル `empty.notenest` を作る
2. NoteNest で開き、エラーメッセージが表示されること、アプリがクラッシュしないことを確認する
3. 同様に `not json` だけ書いたファイルを開き、同じくエラーメッセージが表示されることを確認する

### §20-5 ノート間リンクの整合性

1. ノートを 3 件作成し、ノート A の本文に `[[ノート B]]` と `[[存在しない名前]]` を書く
2. ノート A を保存し、再読込してリンクが残っていることを確認する
3. `Ctrl+Enter` で `[[ノート B]]` 上からノート B へジャンプできることを確認する
4. `[[存在しない名前]]` 上で `Ctrl+Enter` を押すと「リンク先なし」メッセージが表示されることを確認する
5. ノート B の名前を変更し、`[[ノート B]]` がリンク切れ扱いになることを確認する（自動更新されない仕様）

### §20-6 タスク関連ノートの整合性

1. ノートを 2 件作成し、タスクの「関連ノートを設定」でノート 1 を関連付ける
2. タスク項目に 🔗 アイコンが表示されることを確認する
3. 保存して NoteNest を再起動する
4. 同ファイルを開き、関連ノートが維持されていることを確認する
5. ノート 1 を削除する
6. 該当タスクから 🔗 アイコンが消え、関連ノートが自動でクリアされていることを確認する

### §20-7 エクスポートの安全性

1. プロジェクト全体を `.txt` にエクスポートし、メモ帳で開いて日本語が文字化けしないことを確認する
2. ノートブックごとのエクスポートで、フォルダの選択ダイアログをキャンセルする
3. 出力フォルダにファイルが作られていないことを確認する
4. 書き込み禁止フォルダ（例：`C:\Windows`）を選択してエラーメッセージが表示されることを確認する
5. ファイル名に `/` を含むノートブック名でエクスポートし、`_` に置換されることを確認する

### §20-8 ライト/ダークテーマでの主要 UI

1. ライトテーマで以下が読めることを確認する：左ペイン、エディタ、右ペイン、メニュー、ステータスバー、マーカー色、コンテキストメニュー
2. ダークテーマに切り替えて同じ確認を行う
3. テーマ切り替え後にアプリが再描画され、未再起動で反映されることを確認する
4. ノートリンク挿入ダイアログ・ノート選択ダイアログがダークテーマでも読めることを確認する（OS ネイティブ色のままで問題なし）

## §19 ノートリンク挿入UI改善（v0.8.2）

### §19-0 ノートタイトルの重複禁止

1. ノートを追加し、名前を「会議メモ」とする
2. 同じプロジェクト内に、別のノートブックで「会議メモ」という名前のノートを追加しようとする
3. 「ノート名「会議メモ」は既に使用されています。」エラーメッセージが表示されてノートが追加されないことを確認する
4. 「会議メモ2」など別の名前を入力すると正常に追加できることを確認する
5. 既存のノートを右クリック → 名前変更 で「会議メモ」（別ノートと同名）にリネームしようとする
6. 同様のエラーメッセージが表示されてリネームがキャンセルされることを確認する
7. 同一ノートを同じ名前でリネームしようとすると（タイトル変更なし）エラーにならないことを確認する

### §19-1 エディタ右クリックからの選択式挿入

1. ノートが複数あるプロジェクトを開き、いずれかのノートを選択する
2. 本文のカーソルを任意の位置に置く
3. エディタを右クリック → ノートリンクを挿入... を選択する
4. 「ノートリンク挿入」ダイアログが表示されることを確認する
5. リストに「ノートブック名 / ノート名」形式でノート一覧が表示されることを確認する
6. リンク先のノートをリストから選択し、OK を押す
7. カーソル位置に `[[選択したノート名]]` が挿入されることを確認する
8. 挿入後もエディタにフォーカスが残ることを確認する

### §19-2 ダイアログのキーボード操作

1. ノートリンク挿入ダイアログを開く
2. 矢印キーでリスト項目を移動できることを確認する
3. Enter キーで OK が動作することを確認する
4. ダブルクリックでも OK が動作することを確認する
5. Escape キーまたはキャンセルでダイアログが閉じ、挿入されないことを確認する

### §19-3 左ペインのノート右クリックから挿入

1. ノートが複数あるプロジェクトを開く
2. ノート「A」を中央エディタで開いてカーソルを置く
3. **左ペインのノート「B」**（現在開いていない方）を右クリックする
4. コンテキストメニューに「このノートへのリンクを挿入」が表示されることを確認する
5. 「このノートへのリンクを挿入」をクリックする
6. 以下を確認する：
   - 中央エディタのカーソル位置に `[[B]]` が挿入される
   - **エディタのノートは「A」のままで「B」に遷移しない**
   - 左ペインの選択もノート「A」のままであること

### §19-4 タスクコメント編集中の無効化

1. タスクをダブルクリックしてタスクコメント編集モードに入る
2. エディタを右クリックして「ノートリンクを挿入...」を確認する
3. 項目がグレーアウト（無効化）されていることを確認する
4. 左ペインのノートを右クリック → 「このノートへのリンクを挿入」をクリックする
5. 「タスクコメント編集中はノートリンクを挿入できません。」のメッセージが表示されることを確認する
6. 挿入が行われないことを確認する

### §19-5 既存のリンクジャンプが壊れていないことの確認

1. `[[ノート名]]` を含む本文でカーソルをリンク内に置く
2. Ctrl+Enter でリンク先ノートへジャンプすることを確認する
3. エディタ右クリック → ノートリンクを開く でもジャンプできることを確認する
4. リンク先が存在しない場合は「リンク先なし」メッセージが表示されることを確認する

## §18 統合エクスポート

### §18-1 プロジェクト全体の出力

1. ファイル → エクスポートを開く
2. 対象を「プロジェクト全体」、形式を `.txt` にして出力する
3. 全ノートブック・全ノートの内容がUTF-8で出力されることを確認する

### §18-2 対象・形式・付加情報の切替

1. 対象を「現在のノートブック」「現在のノート」に切り替え、選択範囲だけが出力されることを確認する
2. `.md` と `.html` を選択し、各拡張子と内容で出力されることを確認する
3. 「タスクを含める」「マーカー一覧を含める」のON/OFFが出力へ反映されることを確認する
4. 現在のノート／ノートブックを対象にした場合、対象内ノートに関連付けられたタスクだけが出力され、無関係なタスクや未関連タスクが含まれないことを確認する
5. 対象に応じて保存ダイアログの既定ファイル名がプロジェクト名／ノートブック名／ノート名になることを確認する
6. ダイアログまたは保存先選択をキャンセルした場合、ファイルが作成されないことを確認する

## §21 v1.1.0 タスク編集ボタン・マーカーソート切替

### §21-1 タスク ✏ ボタンからのコメント編集

1. 任意のタスクを追加する
2. タスク行右端に ✏ ボタンが表示されていることを確認する
3. ✏ ボタンをクリックする
4. エディタがタスクコメント編集モードに切り替わることを確認する
5. コメントを入力して別のノートを選択し、コメントが保存されることを確認する
6. 再度 ✏ ボタンをクリックすると、保存済みのコメントが表示されることを確認する

### §21-2 右クリックメニュー「コメントを編集...」

1. 任意のタスクを右クリックする
2. コンテキストメニューの先頭に「コメントを編集...」が表示されていることを確認する
3. 「コメントを編集...」を選択する
4. エディタがタスクコメント編集モードに切り替わることを確認する

### §21-3 既存のダブルクリック編集の回帰確認

1. 任意のタスクのタイトル部分をダブルクリックする
2. エディタがタスクコメント編集モードに切り替わることを確認する（従来動作が維持されていること）

### §21-4 マーカーソート切替の動作

前提：複数のノートにまたがる `[TODO]` `[FIXME]` `[NOTE]` が複数行存在すること

1. マーカー一覧フィルタ行の右端にソート選択 ComboBox が表示されていることを確認する
