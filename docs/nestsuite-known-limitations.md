# NestSuite 既知の制約（v1.15.0）

NestSuite の現時点での到達点と、今後の整理対象となっている制約を記録します。

---

## 現在の到達点

- NoteNest / ChatNest / IdeaNest の 3 ツールを 1 つのシェル上でタブ管理できます
- 各ツールは複数ファイルを別タブとして並行利用できます
- タブはファイル／作業単位で独立しており、内容が他のタブに混ざることはありません
- 同じファイルの二重オープンを検出し既存タブへ移動します
- 名前を付けて保存で他タブが開くファイルへの上書きを防ぎます
- 終了時に保存済みファイルタブを記録し、次回引数なし起動時に復元します（v1.15.0）

---

## 制約一覧

### 起動

| 制約 | 状況 |
|------|------|
| ファイル関連付けの自動設定は未対応 | `.notenest` / `.chatnest` / `.ideanest` のファイル関連付けは手動設定が必要 |
| `--classic-notenest` ルートは限定的互換ルートとして維持 | v1.12.0 で縮退方針を整理済み。緊急退避ルートとして当面残すが恒久保守対象ではない。新機能の反映は原則しない。v1.13.0 以降で縮退を実施するか判断する。詳細は [docs/nestsuite-default-startup-plan.md](nestsuite-default-startup-plan.md) を参照 |

### タブ管理

| 制約 | 状況 |
|------|------|
| タブ復元は保存済みファイルのみ対応（v1.15.0） | 未保存タブ・カーソル位置・ペイン状態は復元されない。ファイル指定起動時は復元しない |
| 複数ファイル一括オープンは未対応 | ファイルダイアログで複数選択しても 1 ファイルのみ開く |
| タブ並び替えの本格実装は未対応 | ドラッグ&ドロップによる並び替えは未実装 |

### ファイル形式

| 制約 | 状況 |
|------|------|
| 共通プロジェクトファイル形式は未導入 | `.notenest` / `.chatnest` / `.ideanest` はそれぞれ独立した形式 |
| NoteNest 保存スキーマは 1.4.1 で固定 | 形式変更予定なし |

### アーキテクチャ

| 制約 | 状況 |
|------|------|
| NoteNest は暫定的にタブごとに `MainViewModel` を持つ | 将来的に `NoteNestWorkspaceSessionViewModel` への軽量化を検討 |
| NoteNest 単体版 `MainWindow` は限定的互換ルートとして残す | `--classic-notenest` による起動は可能。ただし恒久保守対象ではなく、新機能は反映しない（v1.12.0 方針） |

---

## 今後の整理候補

以下は将来バージョンで検討する項目です。現時点では実装しません。

- 複数ファイル一括オープン
- NestSuite 用インストーラーおよびファイル関連付け自動設定
- `NoteNestWorkspaceSessionViewModel` への切り出し
- UI モダン化
