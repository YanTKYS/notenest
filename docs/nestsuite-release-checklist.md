# NestSuite リリース前確認チェックリスト（v1.18.0）

NestSuite 統合版のリリース前に確認する項目を整理します。v1.13.0 より、NestSuite を主対象とし、`--classic-notenest` の確認は限定的互換ルートのスモーク確認として扱います。

---

## 1. ビルド・テスト

- [ ] `dotnet build` が通る
- [ ] `dotnet test` が通る（全テスト）
- [ ] `ApplicationVersionTests` でバージョンが正しく表示されることを確認
- [ ] `NoteNest保存スキーマが 1.4.1` のままであることを確認

---

## 2. 起動確認（NestSuite 主対象）

- [ ] `NoteNest.exe` → NestSuite として起動する（無題 NoteNest タブ）
- [ ] `NoteNest.exe sample.notenest` → NestSuite で `.notenest` タブを開く
- [ ] `NoteNest.exe sample.chatnest` → NestSuite で `.chatnest` タブを開く
- [ ] `NoteNest.exe sample.ideanest` → NestSuite で `.ideanest` タブを開く
- [ ] `NoteNest.exe --nestsuite` → NestSuite として起動する（互換フラグ、フラグなしと同じ動作）
- [ ] `NoteNest.exe --nestsuite sample.notenest` → NestSuite で `.notenest` タブを開く

### `--classic-notenest` スモーク確認（限定的互換ルート）

- [ ] `NoteNest.exe --classic-notenest` → NoteNest 単体版として起動できる（起動不能でない）
- [ ] `NoteNest.exe --classic-notenest sample.notenest` → 指定ファイルを開ける

---

## 3. NestSuite 基本操作

### タブ作成

- [ ] NoteNest タブを新規作成できる
- [ ] ChatNest タブを新規作成できる
- [ ] IdeaNest タブを新規作成できる
- [ ] NoteNest タブを複数同時に開ける
- [ ] ChatNest タブを複数同時に開ける
- [ ] IdeaNest タブを複数同時に開ける

### ファイルを開く

- [ ] `.notenest` ファイルを開ける
- [ ] `.chatnest` ファイルを開ける
- [ ] `.ideanest` ファイルを開ける
- [ ] 同じ `.notenest` を再度開くと既存タブがアクティブになる
- [ ] 同じ `.chatnest` を再度開くと既存タブがアクティブになる
- [ ] 同じ `.ideanest` を再度開くと既存タブがアクティブになる

### 複数ファイル一括オープン（v1.16.0）

- [ ] 「開く...」ダイアログで複数ファイルを同時選択できる
- [ ] `.notenest` / `.chatnest` / `.ideanest` を混在して複数選択し、それぞれタブとして開ける
- [ ] 既に開いているファイルを含めても重複タブにならない（既存タブがアクティブになる）
- [ ] 一部のファイルが存在しない場合、他のファイルは開き、「一部のファイルを開けませんでした」が表示される
- [ ] 開いたファイルが最近使ったファイルに登録される

### 最近使ったファイル（v1.14.0 / v1.14.1）

- [ ] ファイルメニューに「最近使ったファイル」サブメニューが表示される
- [ ] ファイルを開くと最近使ったファイルの先頭に追加される（3 ツール横断）
- [ ] 同じファイルを再度開くと重複せず先頭に移動する
- [ ] 最近使ったファイルから既存タブのファイルを選ぶと既存タブがアクティブになり、履歴でも先頭へ移動する（v1.14.1）
- [ ] 「開く...」から既存タブのファイルを選んだ場合も、履歴の先頭へ移動する（v1.14.1）
- [ ] 最近使ったファイルから削除・移動済みのファイルを選ぶとエラーダイアログが表示され履歴から削除される
- [ ] 最近使ったファイルに未対応拡張子のパスが混入している場合、エラーダイアログが表示され履歴から削除される（v1.14.1）
- [ ] NoteNest タブで「名前を付けて保存」すると保存先が最近使ったファイルに追加される
- [ ] アプリ再起動後も最近使ったファイルが引き継がれる

### 保存

- [ ] NoteNest タブを上書き保存できる（メニュー・Ctrl+S 両方）
- [ ] ChatNest タブを上書き保存できる（メニュー・Ctrl+S 両方）
- [ ] IdeaNest タブを上書き保存できる（メニュー・Ctrl+S 両方）
- [ ] 保存後に未保存マーク（`*`）が消える
- [ ] 名前を付けて保存で別タブが開くパスを指定するとエラーが表示される

### Ctrl+S ショートカット（v1.16.1）

- [ ] NoteNest タブで Ctrl+S を押すと上書き保存される
- [ ] ChatNest タブで Ctrl+S を押すと上書き保存される
- [ ] IdeaNest タブで Ctrl+S を押すと上書き保存される
- [ ] 未保存（パス未設定）タブで Ctrl+S を押すと「名前を付けて保存」に進む
- [ ] メニューの「上書き保存」に Ctrl+S が自動表示される
- [ ] メニューの「上書き保存」クリックと Ctrl+S の挙動が一致する

### タブを閉じる

- [ ] 保存済みタブを閉じられる
- [ ] 未保存タブを閉じようとすると確認ダイアログが表示される（タブ名が明示されている）
- [ ] 確認でキャンセルするとタブが残る
- [ ] 確認で破棄するとタブが閉じる
- [ ] 最後のタブを閉じると無題 NoteNest タブが自動作成される

### タブ復元（v1.15.0）

- [ ] 保存済みファイルタブを開いた状態でアプリを終了し、再起動すると前回のタブが復元される
- [ ] `.notenest` / `.chatnest` / `.ideanest` の 3 形式すべてが復元される
- [ ] 前回アクティブだったタブが復元後に選択されている
- [ ] 未保存タブ（無題）は復元されない
- [ ] `NoteNest.exe sample.notenest` など引数指定起動ではタブ復元されない（指定ファイルのみ開く）
- [ ] セッションに記録されたファイルが存在しない場合はスキップして他のファイルを復元する
- [ ] 復元対象が 1 件もない場合は無題 NoteNest タブが開く
