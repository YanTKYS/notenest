# 運用メモ — NoteNest v0.1.0

## 利用時の注意

### データの保全
- **NoteNest はリアルタイム自動保存を行いません**。`Ctrl+S` または メニュー → ファイル → 保存 を定期的に行ってください。
- 保存していない状態でアプリが強制終了した場合、未保存の変更は失われます。
- タイトルバーに「*」が表示されている間は未保存の変更があります。

### ファイルの場所
- `.notenest` ファイルは自分で選択した場所に保存されます。
- ファイルを誤って削除しないよう、重要なプロジェクトは専用フォルダに保存することをお勧めします。

---

## `.notenest` ファイルのバックアップ

### 手動バックアップの手順
1. `.notenest` ファイルを右クリック → コピー
2. バックアップフォルダ（例: `C:\Users\<ユーザー名>\Documents\NoteNest_Backup\`）に貼り付け
3. ファイル名に日付を付けて保存（例: `業務改善プロジェクト_20260531.notenest`）

### 推奨バックアップ周期
- 重要なプロジェクト：毎日
- 通常利用：週 1 回

### 注意
- `.notenest` ファイルは UTF-8 エンコードの JSON テキストです。
- テキストエディタで開いて内容を確認することができますが、**手動での編集は推奨しません**。手動編集による JSON の破損でファイルが開けなくなる場合があります。

## 一般職員に配布する前に確認すべき事項

### 動作確認チェックリスト

- [ ] .NET 8.0 デスクトップランタイムがインストールされているか
- [ ] アプリが正常に起動するか
- [ ] サンプルプロジェクトが表示されるか
- [ ] `.notenest` ファイルの保存・読込が正常に動作するか
- [ ] フォント（Yu Gothic UI など）が正常に表示されるか
- [ ] 高 DPI ディスプレイ環境でレイアウトが崩れないか

### 配布前に推奨する対応

1. **配布形式の確認**
   - 自己完結型（Self-Contained）の実行ファイルとしてビルドすれば、.NET ランタイムのインストールが不要になります。
   - `dotnet publish -r win-x64 --self-contained -c Release` でビルドできます。

2. **初期設定の確認**
   - 初回起動時にサンプルプロジェクトが表示されることを確認する。

3. **ファイル保存場所の案内**
   - 利用者がどこに `.notenest` ファイルを保存するか、ガイドラインを作成することを推奨します。

4. **バックアップ手順の周知**
   - このドキュメントの「バックアップ」セクションを利用者に共有してください。

5. **未保存データの紛失リスクの周知**
   - 自動保存がないため、定期的に `Ctrl+S` で保存する習慣を案内してください。

### サポート窓口
配布後に問題が発生した場合は、以下の情報を収集してください。
- エラーメッセージのスクリーンショット
- 発生したときの操作手順
- 使用している `.notenest` ファイルのコピー（個人情報に注意）
- OS バージョンと .NET バージョン（`dotnet --version` コマンドで確認）
