# 運用メモ — NoteNest v1.0.0

## 利用時の注意

### データの保全
- **NoteNest はリアルタイム自動保存を行いません**。`Ctrl+S` または メニュー → ファイル → 保存 を定期的に行ってください。
- 保存していない状態でアプリが強制終了した場合、未保存の変更は失われます。
- タイトルバーに「*」が表示されている間は未保存の変更があります。
- 未保存状態が 5 分以上続くと、ステータスバーの「⚠ 未保存（N分）」が赤色で表示されます（v0.4.0）。

### 複数ウィンドウ利用時の注意
- NoteNest は複数起動に対応していますが、**同じ `.notenest` ファイルを複数ウィンドウで同時編集しないでください**。
- 後から保存した内容で上書きされるため、一方の変更が失われる場合があります。
- 複数ウィンドウを使う場合は、それぞれ異なる `.notenest` ファイルを開いてください。

### ファイルの場所
- `.notenest` ファイルは自分で選択した場所に保存されます。
- ファイルを誤って削除しないよう、重要なプロジェクトは専用フォルダに保存することをお勧めします。

## `.notenest` ファイルのバックアップ

### 手動バックアップの手順
1. `.notenest` ファイルを右クリック → コピー
2. バックアップフォルダ（例: `C:\Users\<ユーザー名>\Documents\NoteNest_Backup\`）に貼り付け
3. ファイル名に日付を付けて保存（例: `業務改善プロジェクト_20260531.notenest`）

### 推奨バックアップ周期
- 重要なプロジェクト：毎日
- 通常利用：週 1 回

### 注意
- `.notenest` ファイルは UTF-8 エンコードの JSON テキストです。
- テキストエディタで開いて内容を確認することができますが、**手動での編集は推奨しません**。手動編集による JSON の破損でファイルが開けなくなる場合があります。

## `.bak` ファイルからの復元

NoteNest は保存時にアトミック保存を行っており、既存ファイルがあるときは保存前の状態を `.bak` ファイルとして自動的に残します。

### 仕様

- `mydata.notenest` を保存すると、同じフォルダに `mydata.notenest.bak` が作られる（または上書きされる）
- `.bak` には **直前の保存内容** のみが保持される（複数世代の履歴は保持しない）
- 一時ファイル `mydata.notenest.tmp` は保存成功時に自動削除される

### 復元手順

`.notenest` 本体が破損して開けなくなった場合：

1. NoteNest を一旦終了する
2. エクスプローラーで該当フォルダを開く
3. 破損した `mydata.notenest` を別名（例：`mydata.notenest.broken`）に変更して退避する
4. `mydata.notenest.bak` を `mydata.notenest` にリネームする
5. NoteNest を起動して該当ファイルを開く

直近の保存以前の状態に戻りますが、それ以降の編集内容は失われます。重要なファイルは別途バックアップを取ることを強く推奨します。

### `.tmp` ファイルが残っている場合

保存中にアプリが強制終了した場合、`.tmp` ファイルが残る可能性があります。

- `.notenest` 本体が読める場合は、`.tmp` を削除して問題ありません
- 次回保存時に `.tmp` は新しい内容で上書きされます

## テキストエクスポートの利用上の注意

- **エクスポートファイルは外部確認用**です。`.notenest` が正式なプロジェクトファイルであり、エクスポートした `.txt` ファイルを NoteNest に再読込することはできません。
- タスク一覧・タスクコメント・マーカー集計は v0.8.1 の出力対象外です。
- 出力ファイルは UTF-8（BOM 付き）で保存されます。Windows メモ帳で正常に開けます。
- ノートブックごとの出力時は、同名ノートブックに自動で連番が付与されます（例: `会議メモ.txt`、`会議メモ_2.txt`）。

---

## 保存形式の安定性について

NoteNest は v1.0.0 で安定版リリースに到達しました。

- `.notenest` の保存形式（JSON スキーマ）は v1.0.0 以降、後方互換性を保つ方針です
- 新しいフィールドが追加される場合でも、v1.0.0 で保存した `.notenest` を将来のバージョンが読み込めることを目指します
- `version` フィールドは将来のマイグレーション対応のために維持されています
- 重大なスキーマ変更が必要になる場合は、メジャーバージョンを上げ、マイグレーション処理を提供します

## v1.0.0 時点の制限

### エディタの制限
| 機能 | 状態 | 備考 |
|------|------|------|
| Markdown プレビュー | **未対応** | v0.7.2 以降で検討 |
| シンタックスハイライト | **未対応** | v0.7.2 以降で検討 |
| 行番号表示 | **対応済み**（v0.5.0、編集メニュー → 行番号を表示） | 折り返し行では行番号位置にずれが生じる場合あり |
| 元に戻す（Undo） | **OS 標準のみ** | `Ctrl+Z` は TextBox の標準機能で動作 |

| テーマ切り替え | **対応済み**（v0.7.2、編集メニュー → ダークテーマ） | メニューバー・スクロールバー・ダイアログは OS ネイティブ色のまま |
| ノート間リンク | **対応済み**（v0.8.0、`[[ノート名]]` 記法 + Ctrl+Enter でジャンプ；v0.8.2 でリンク挿入を選択式に改善） | リンクの視覚的ハイライト・バックリンク集計は未対応 |

### タスク管理の制限
| 機能 | 状態 |
|------|------|
| 期限設定 | **未対応** |
| 優先度設定 | **未対応** |
| 担当者設定 | **未対応** |
| タスクのドラッグ並べ替え | **対応済み**（v0.5.0 グループ内、v0.6.0 グループ間移動） |
| グループ間のタスク移動 | **対応済み**（右クリック→グループを変更、またはドラッグ＆ドロップ） |
| タスクとノートの関連付け | **対応済み**（v0.8.0、右クリック→関連ノートを設定） |

### ノート間リンクの制限
- **リンク先解決はタイトルベース**。`[[ノート名]]` は実行時にタイトルで検索するため、ノート名を変更すると既存の `[[旧ノート名]]` リンクは自動更新されずリンク切れになります。
- **同名ノートが複数ある場合**（v0.8.2 以前に作成したプロジェクトや `.notenest` を手動編集したケースなど）、`[[ノート名]]` は最初に見つかったノートへ解決されます。v0.8.2 以降、ノート追加・リネームで同名ノートを新規作成できなくなりました。既存データに重複がある場合、リンク挿入時に警告を表示します。
- **大文字小文字の区別なし**。`[[会議メモ]]` と `[[会議めも]]` は別扱いですが、`[[Note A]]` と `[[note a]]` は同一とみなされます。

### ノート管理の制限
| 機能 | 状態 |
|------|------|
| ノートブック内でのノートの並べ替え | **対応済み**（v0.2.0、右クリック→上に移動/下に移動） |
| ノートブックの並べ替え | **対応済み**（v0.2.0、右クリック→上に移動/下に移動） |
| ノートブック間でのノート移動 | **対応済み**（v0.6.0、ドラッグ＆ドロップ） |

### マーカーの制限
| 機能 | 状態 |
|------|------|
| 全ノート横断マーカーの集計 | **対応済み**（v0.1.1、右下ペイン最下部に表示） |
| マーカー種別フィルタ | **対応済み**（v0.2.0、TODO/FIXME/NOTE のチェックボックス） |
| 全ノート横断マーカーのクリックナビゲーション | **対応済み**（v0.3.0） |
| マーカークリック時の左ツリー選択同期 | **対応済み**（v0.4.0） |
| カスタムマーカーキーワード | **未対応**（`[TODO]` `[FIXME]` `[NOTE]` 固定） |
| マーカー行の表示／非表示 | **未対応**。WPF 標準 TextBox では行単位の非表示を安全に実装しにくく、本文消失・保存不整合リスクがあるため未実装。詳細は `docs/backlog.md` を参照。 |
| 入力時の全体集計パフォーマンス | 本文編集のたびに全ノート本文を再スキャンするため、プロジェクト規模が大きい場合（ノート数 100 件・各数十 KB 以上）は入力遅延が生じる可能性があります。通常規模では問題になりません。 |

### 今バージョンで実装していない機能
| 機能 | 理由 |
|------|------|
| 画像貼り付け | NoteNest は軽量テキスト管理ツールのため |
| 共同編集 | ローカル単一ファイル管理の思想と合わないため |
| 文字数表示 | 現時点の主要価値ではないため |

### プロジェクトの制限
- 1 ウィンドウにつき 1 プロジェクトのみ（複数プロジェクトは別ウィンドウで起動して利用）
- 自動保存は未実装です

### 配布前に推奨する対応

1. **配布形式の確認**
   - 自己完結型（Self-Contained）の実行ファイルとしてビルドすれば、.NET ランタイムのインストールが不要になります。
   - `dotnet publish -r win-x64 --self-contained -c Release` でビルドできます。

2. **初期設定の確認**
   - 初回起動時にサンプルプロジェクトが表示されることを確認する。

3. **ファイル保存場所の案内**
   - 利用者がどこに `.notenest` ファイルを保存するか、ガイドラインを作成することを推奨します。

4. **バックアップ手順の周知**
   - このドキュメントの「バックアップ」セクションを利用者に共有してください。

5. **未保存データの紛失リスクの周知**
   - 自動保存がないため、定期的に `Ctrl+S` で保存する習慣を案内してください。

### サポート窓口
配布後に問題が発生した場合は、以下の情報を収集してください。
- エラーメッセージのスクリーンショット
- 発生したときの操作手順
- 使用している `.notenest` ファイルのコピー（個人情報に注意）
- OS バージョンと .NET バージョン（`dotnet --version` コマンドで確認）
